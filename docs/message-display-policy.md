# メッセージ表示方針

## 基本方針

ユーザーの入力内容だけで判断できるバリデーションはフロントエンドで表示する。

サーバー側の判定が必要な内容はバックエンドで判定し、API連携後に表示方法を整理する。

## フロントで表示するもの

- 必須入力
- メールアドレス形式
- 郵便番号形式
- 電話番号形式
- パスワード文字数
- パスワード確認の一致
- 日付未選択
- 時間未選択
- 数量の下限

## バックエンドで判定するもの

- ログイン情報の正誤
- メールアドレスの重複
- 在庫不足
- 商品の存在確認
- 注文可否
- 権限エラー
- セッション切れ

## 実装方針

- 共通文言は `src/lib/messages.ts` に定義する。
- 入力欄下のエラーは `FieldError` を使う。
- フォーム上部や処理結果のメッセージは `Message` を使う。
- APIレスポンスの `message` をそのまま表示するかはAPI連携時に再検討する。
